Interweb

TCP

TCP staat voor Transmission Control Protocol. Het werkt door een verzoek om te verbinden te sturen. Daarna stuurt de server een erkenning. Waarna de computer weer een erkenning terug
stuurt. Nu is er een verbinding.

definition@source:~$ Wikipedia

UDP

UDP staat voor User Datagram Protocol. Het lijkt een beetje op TCP, maar het heeft niet de connectie stap die TCP wel heeft. UDP stuurt heel simpel gewoon de data door naar het IP dat is opgegeven zonder eerst te proberen te verbinden.

definition@source:~$ Wikipedia

IPv4

IPv4 staat voor Internet Protocol version 4. Het is een protocol om te zorgen dat je met andere computers kan verbinden over een WAN of LAN netwerk.

definition@source:~$ Ome Dennis

IPv6

IPv6 staat voor Internet Protocol version 6. Het is een protocol om te zorgen dat je met andere computers kan verbinden over een WAN of LAN netwerk.

definition@source:~$ Ome Dennis

IP adres

IP adress is het Internet protocol adress. Dit zit er vaak uit als het volgende '127.0.0.1' (IPv4). Het is bedoeld als een soort ID voor als je op het internet bent zodat servers weten waar een verzoek vandaan komt en waar ze een hun antwoord naartoe moeten sturen.

definition@source:~$ Ome Dennis

Subnetmask

een subnetwerk is een netwerk dat in twee of meer delen is gedeeld. een subnetmask is er om aan te geven welke pc bij welk netwerk hoort.

definition@source:~$ Wikipedia

Hardware

Hub

Een hub is een apparaat dat 1 netwerk poort splitst naar meerdere poorten.
Vaak 4, 8, 16, 32 of zelfs 64 poorten.

definition@source:~$ Wikipedia

Switch

Een switch is bijna hetzelfde als een hub, maar een hub heeft geen logica en werkt verder als het originele netwerk.
een switch een subnet maakt.

definition@source:~$ Wikipedia

Bridge

Een bridge is een apparaat wat 2 delen van een netwerk aan elkaar kan linken.

definition@source:~$ Wikipedia

Router

Een router is een apparaat die data verstuurd tussen 2 netwerken.

definition@source:~$ Wikipedia

OSI Model

Applicatie Laag

De applicatie laag is de laag die het dichts bij de end-user komt. Dit zijn de bytestreams en binary arrays die je binnen krijgt over het netwerk

definition@source:~$ Wikipedia

Presentatie Laag

De presentatie laag transformeerd de data vanuit de sessie laag naar data wat de applicatie laag kan begrijpen.
Dus de raw bits omzetten naar een bytestream of een binary array

definition@source:~$ Wikipedia

Sessie Laag

De sessie laag zorgt voor de dialoog tussen de computers. Bijvoorbeeld hoeveel packet's nog binnen moeten komen etc.

definition@source:~$ Wikipedia

Transport Laag

De transport laag is de laag die alles regelt met content van variabele lengte, zoals VoIP (Voice over Internet Protocol)

definition@source:~$ Wikipedia

Netwerk Laag

De netwerk laag is de laag die ervoor zorgt dat "packets" van een netwerk node naar de andere komt

definition@source:~$ Wikipedia

Datalink Laag

De datalink laag is de laag die fouten kan vinden in de fysieke laag.
Bijvoorbeeld het detecteren van bitshifts.

definition@source:~$ Wikipedia

Fysieke Laag

De fysieke laag is de laag die "rauwe" data stuurt. En dus digitale bit's om zet naar electrische signalen.

definition@source:~$ Wikipedia
